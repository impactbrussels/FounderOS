# Founder OS - Skills Roadmap

> The open operating system for first-time founders. This roadmap is the contribution catalogue.

Founder OS ships **23 skills** today and is built to grow. Every **planned slot** below is a
real founder job that the OS does not yet cover - and an open invitation. If you have run a
startup, advised one, or simply solved one of these problems with AI, you can author the skill
that teaches the next founder how to do it.

## How to claim a slot

1. Find a slot below marked 🟢 / 🟡 / 🔵 (not ✅ Shipped).
2. Open a **New skill** issue (`.github/ISSUE_TEMPLATE/new_skill.yml`) and name the slot you're
   claiming so two people don't build the same thing.
3. A maintainer assigns it; you fork, author, and open a PR.
4. Full instructions live in **[CONTRIBUTING.md](CONTRIBUTING.md)**. The gold-standard example
   to copy is **[`skills/validate-idea/`](skills/validate-idea/)**.

## Status legend

| Status | Meaning |
|--------|---------|
| ✅ **Shipped** | Live in `skills/`, linted, documented |
| 🟢 **good first issue** | Well-scoped, low-context - a great first contribution |
| 🟡 **help wanted** | Real method exists, needs an experienced author |
| 🔵 **design needed** | Scope or method still open - propose the shape in your issue |

## The five founder problem-types

The planned slots are organised around the five things founders actually ask an AI co-pilot to
do, restated for startups from the nonprofit AI prompt cookbook's problem-type taxonomy:

| Problem-type | For a founder, this means |
|--------------|---------------------------|
| **Create & Communicate** | Draft the deck, the page, the post, the email, the script |
| **Automate Admin** | Turn a repeated manual chore into a repeatable, low-touch workflow |
| **Answer & Assist** | Stand up a way to answer customers, investors, and teammates fast |
| **Sort & Scan** | Triage, segment, prioritise, and screen a pile of inputs |
| **Learn & Decide** | Turn evidence into a clear, defensible go / no-go call |

Each planned skill below names its problem-type in the **Function** column so contributors can
pick work that fits their strength.
