# FounderOS Launch Comms Pack

Everything to announce FounderOS across channels. Written in Adam's voice. Replace `REPO_URL`
with the live link before posting. Guiding rule: a project called *proof before hype* earns
attention with substance, not volume.

| File | Channel | Use |

## Launch sequence (suggested 1 week)

**Day 0 — go live**
- Repo public; social preview image uploaded (see [../brand/social-preview-brief.md](../brand/social-preview-brief.md)); Discussions on; 3–5 `good first issue`s opened; repo pinned to your profile.

**Day 1 — the story**
- **LinkedIn:** post the long story (launch-post.md). Best mid-morning your audience's time.
- **X:** post the thread the same day, a few hours apart.
- **Substack:** post **Note 1** (announcement).

**Day 2–3 — the why**
- **Substack:** Note 2 (origin) or Note 3 (belief problem → ties to *Proof Before Hype*).
- **X/LinkedIn:** reply to comments; reshare the thread's best tweet as a standalone.

**Day 4–5 — give value**
- **Substack:** Note 4 (validate-idea tactic) or Note 5 (first-vs-fifth purchase).
- Consider a short LinkedIn post teaching ONE skill in full (pick the skill's copy-paste prompt and show the output). Value posts outperform announcements.

**Day 6–7 — the movement**
- **Substack:** Note 6 (anti-gatekeeping) + Note 8 (call for contributors).
- **LinkedIn:** a follow-up post: "X stars, Y contributors in a week — here's what people are building." (Social proof, only if real.)

**Ongoing**
- A FoodEdge Substack long essay tying FounderOS to your books and the food-founder audience; link Note 3 or 6 to it.
- When you enroll in GitHub Sponsors (see [../SPONSORS-SETUP.md](../SPONSORS-SETUP.md)), add a soft sponsor mention.

## Cross-promo rules

- **One ask per post.** Star the repo, or read the book, or subscribe — not all three.
- **Books live in the bio/PS, not the headline.** Let the OS earn the click; the book is the
  "if this resonates" follow-through (the credible way to feature it).
- **Tailor, don't copy-paste across channels.** LinkedIn = story; X = punch; Substack = idea + pull.
- **Engage > broadcast.** Replying with a useful FounderOS tactic beats another announcement.
- **Stay on-creed.** No hype words, no fake urgency. Show the working thing.
