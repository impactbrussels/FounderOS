# Early Sales Playbook (reference)

Frameworks behind the `find-first-customers` skill. Theme-agnostic; the founder supplies the domain.

## Where to source the first list (named, reachable people)

- **Your network's network** - who do you know who knows an `[ICP]`? Ask for warm intros by name.
- **Communities** - Slack/Discord groups, subreddits, forums, LinkedIn groups where `[ICP]` gathers.
- **Search** - LinkedIn/Sales Nav, job titles, "[industry] + [role]", local directories.
- **Competitor & alternative reviews** - people complaining about the tool/workaround they use today are pre-qualified.
- **Events** - attendee/speaker lists from conferences your `[ICP]` attends.
- **Content engagers** - people who commented on a relevant post about `[PROBLEM]`.

Aim for 30-50 to start. For each: name, where you found them, and a one-line *why this person*.

## Outreach message templates

### Cold email - GOOD

> Subject: quick question about [specific situation]
>
> Hi [Name] - saw [specific, true reason you picked them: their post / role / a review they left].
> I'm building something for [ICP] who [PROBLEM in their words]. I'm not selling - I'd love 15
> minutes to learn how you handle [PROBLEM] today. Worth a quick call this week?
> - [Founder], [STARTUP_NAME]

Why it works: personalized opener, names *their* problem, asks for time not money, ≤5 sentences.

### Cold email - BAD

> Subject: Revolutionizing [industry] with AI 🚀
>
> Hi there, I hope this email finds you well! We're an innovative, cutting-edge platform that
> leverages synergistic solutions to 10x your [everything]. Our best-in-class product has many
> features including X, Y, Z, A, B, C. Can I book 30 minutes to walk you through a full demo?

Why it fails: no personalization, about *us*, feature dump, big ask, hype words, too long.

### DM / LinkedIn - GOOD

> Hi [Name], I noticed [specific true detail]. I'm talking to [ICP] about how they deal with
> [PROBLEM] - not pitching, just learning. Open to a 15-min chat?

Keep DMs even shorter than email. No links in the first message - it reads as spam.

## Simple pipeline / CRM table

A spreadsheet is enough for the first 50. Track one row per person:

| Name | Source / why them | Channel | Stage | Last touch | Next step + date | Notes / reason for no |
|------|-------------------|---------|-------|-----------|------------------|-----------------------|
| | | | Contacted | | | |
| | | | Replied | | | |
| | | | Call booked | | | |
| | | | Trial / pilot | | | |
| | | | Paid | | | |

Stages: **Contacted → Replied → Call → Trial → Paid.** Your conversion `[METRIC]` is the drop-off
between stages - that's where you learn whether to fix the list, the message, or the product.

## Qualifying questions (disqualify fast - your time is scarce)

- "Walk me through how you handle [PROBLEM] today." (Confirms the problem is real and current.)
- "How much does that cost you - time, money, or headaches?" (Frequency × intensity = willingness to pay.)
- "Who else is involved when you decide on something like this?" (Finds the real buyer.)
- "If this solved [PROBLEM], what would change for you?" (Surfaces the value they'd pay for.)
- "Is this a now problem or a someday problem?" (Disqualifies the not-yet.)

If they don't have the problem, or won't prioritize it, thank them and move on - fast.

## Objection-handling table (respond honestly, don't manipulate)

| Objection | What's really going on | Honest response |
|-----------|------------------------|-----------------|
| "Too expensive" | Value not yet clear, or wrong buyer | Re-anchor on the cost of the problem; if still no, you may have the wrong `[ICP]` |
| "No time right now" | Low priority, or polite no | "When does [PROBLEM] become urgent for you?" - if never, disqualify |
| "Send me info" | Soft brush-off | Offer a 15-min call instead; one-pager only if they name a real next step |
| "We already use [X]" | Has an alternative | Ask what [X] *doesn't* do; if nothing, they're not your customer yet |
| "Let me think about it" | Unspoken doubt | "What would you need to see to feel confident?" - name the real blocker |
| "Not the decision-maker" | Wrong contact | Ask for a warm intro to the person who is |

Log the **reason** every time. Three "too expensive" in a row is a pricing or positioning signal,
not three coincidences.

## "Do things that don't scale" tactics

- **Hand-onboard** every early user yourself; do the setup *for* them if needed.
- **Concierge the value** - deliver the outcome manually before the product fully does it.
- **Over-respond** - answer in minutes, not days; founders out-care incumbents.
- **Personal follow-up** - a real note beats any drip campaign at this stage.
- **Recruit from conversations** - every call ends with "who else has this problem?"

These don't scale - that's the point. They buy you customers *and* learning you can't get from automation.

## Anti-patterns

- **Spray-and-pray automation** - blasting a templated sequence to 1,000 strangers before you've closed one by hand.
- **Pitching before listening** - leading with the demo instead of confirming the problem.
- **Refusing to ask for money** - endless "feedback chats" that never make the ask. Earn the right, then ask.
- **Arguing with a "no"** - trying to win the objection instead of logging the reason.
- **Selling to the unqualified** - chasing polite-but-uninterested people because they reply.
- **No pipeline** - running outreach from memory, so you can't see your real conversion or learn from it.
- **Waiting for inbound** - building features and hoping customers appear. At this stage, you go to them.

## Further reading (verify against primary sources)

*The Mom Test* (Rob Fitzpatrick) · "Do Things That Don't Scale" (Paul Graham essay) ·
*The Sales Acceleration Formula* (Mark Roberge) · *Founding Sales* (Pete Kazanjy). Treat these as
method sources - what your actual `[ICP]` says on a call beats any script.

**Frameworks from the author's books (generalized):**

- **First Purchase vs Fifth Purchase** - trial (curiosity, cheap) and repeat (habit/belief, expensive) are different problems; design for both. From *Proof Before Hype* by Adam M. Adamek.
